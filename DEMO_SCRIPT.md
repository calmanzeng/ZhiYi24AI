# 30秒 Demo 演示脚本

## 准备
- [ ] 打开电脑 + 外接摄像头（或笔记本内置）
- [ ] 运行: python demo.py
- [ ] 确保窗口正常显示

## 演示流程

### 第一步: 正常 CPR（15秒）
→ 对着摄像头，双手叠放在胸前，按 100-120/分的节奏做 CPR
→ 观察: 分数应该在 80-100，标记为绿色或 A 级

**说**: "这是标准 CPR — AI 实时识别手腕运动轨迹，计算按压频率和节奏一致性。"

### 第二步: 故意做快（5秒）
→ 疯狂快速按压（150+/分）
→ 观察: 分数骤降，反馈显示 "TOO FAST"

**说**: "按压太快 — 系统立刻检测到并给出红色警告。"

### 第三步: 故意做慢（5秒）
→ 慢慢悠悠地按压（60/分）
→ 观察: 分数低，反馈显示 "TOO SLOW"

**说**: "按压太慢 — 同样能被实时捕捉。"

### 第四步: 节奏乱（5秒）
→ 随机节奏，忽快忽慢
→ 观察: 一致性 CV 值飙升

**说**: "甚至能够检测节奏是否均匀 — 这是人类考官很难精确判断的。"

### 总结
**说**: "这个 MVP 用了 4 小时搭建，零成本。扩展到体格检查、缝合打结、气管插管 — 每一项都可以用同样的 AI 评分思路。你们的硬件 + 我们的 AI = 没有竞争对手。"
